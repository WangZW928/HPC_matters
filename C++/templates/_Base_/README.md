# C++ 模板基础笔记

这个小项目回答几个核心问题：什么是模板、模板如何编译和实例化、函数模板如何做类型推导、非类型模板参数有什么用，以及 `Array<int, 16>` 的内存到底在哪里。

## 1. 什么是模板

模板是 C++ 的“代码生成机制”。你写一份通用代码，编译器根据具体类型或常量参数生成具体版本。

例如：

```cpp
template <typename T>
T add(T a, T b) {
    return a + b;
}
```

当你调用：

```cpp
add<int>(1, 2);
add<double>(1.5, 2.5);
```

编译器会分别生成类似 `int add(int, int)` 和 `double add(double, double)` 的函数版本。

## 2. 模板的编译流程

模板本身不是最终机器码。它更像一张“生成代码的图纸”。

典型流程：

1. 编译器先解析模板定义，检查基础语法。
2. 遇到具体使用点，例如 `Array<int, 16>`。
3. 编译器把模板参数代入模板定义。
4. 生成一个具体类型或函数，这一步叫实例化。
5. 对实例化后的具体代码做类型检查、优化、生成机器码。

所以模板错误经常出现在使用点附近，因为很多检查必须等到具体类型出现后才能完成。

## 3. 模板的使用和实例化

模板的使用分为两类：

```cpp
Array<int, 16> arr;
```

这是类模板实例化。`T = int`，`Size = 16`，编译器生成一个具体类型 `Array<int, 16>`。

```cpp
add(1, 2);
```

这是函数模板实例化。编译器根据实参 `1` 和 `2` 推导出 `T = int`，再生成 `int add(int, int)`。

注意：`Array<int, 16>` 和 `Array<int, 32>` 是两个不同类型，因为模板参数不同。

## 4. 什么是模板导出

早期 C++ 标准曾经有一个 `export template` 机制，目标是把模板声明和模板定义分离到不同文件里。

但它有几个问题：

- 编译器实现复杂。
- 支持它的编译器很少。
- 实际工程里几乎不用。

后来标准移除了传统意义上的模板导出。现代 C++ 中，你通常应该把模板定义放在头文件里。

## 5. 类模板成员函数通常以内联方式实现，怎么理解

这句话容易误解。它通常不是说“编译器一定会把函数调用展开成内联机器码”，而是说：

类模板的成员函数定义通常直接写在头文件或类定义附近，让每个使用它的翻译单元都能看到完整定义。

例如：

```cpp
template <typename T>
struct Box {
    T value;

    T get() const {
        return value;
    }
};
```

这里 `get()` 写在类模板内部，所以它天然是 inline 函数定义形式。这样做的主要原因是：模板实例化时，编译器必须看到函数体。

重点：

- “写在类里”使它具有 inline 语义，允许多个翻译单元包含同一份定义。
- “inline 语义”主要解决链接层面的重复定义问题。
- 是否真的做函数调用展开，是优化器决定的，不由 `inline` 关键字绝对保证。

## 6. 函数模板的语法怎么理解

这句话：

> 函数模板的语法与类模板基本相同，也是以关键字 template 和模板参数列表作为声明与定义的开始。模板参数列表中的类型，可以出现在参数、返回值以及函数体中。

可以拆开看：

```cpp
template <typename T>
T max_value(T a, T b) {
    T result = a < b ? b : a;
    return result;
}
```

这里的 `T` 来自模板参数列表：

```cpp
template <typename T>
```

然后 `T` 出现在三个位置：

- 返回值：`T max_value(...)`
- 函数参数：`T a, T b`
- 函数体：`T result`

意思是：`T` 不是某个固定类型，而是一个待编译器替换的类型占位符。

## 7. 编译器的类型推导

函数模板通常可以不手写模板参数：

```cpp
add(1, 2);       // T 推导为 int
add(1.5, 2.5);   // T 推导为 double
```

编译器会根据函数实参类型推导模板参数。

但推导也有限制：

```cpp
add(1, 2.5);     // 可能失败，因为 T 同时像 int 又像 double
```

这时可以显式指定：

```cpp
add<double>(1, 2.5);
```

类模板在 C++17 之前通常需要显式写模板参数，例如 `Array<int, 16>`。C++17 之后有 CTAD（类模板参数推导），但不是所有类模板都能自动推导，具体取决于构造函数和推导指引。

## 8. 模板的整型参数是干啥的

`template <typename T, int Size>` 中的 `Size` 叫非类型模板参数。

它不是类型，而是一个编译期常量值：

```cpp
template <typename T, int Size>
struct Array {
    T data[Size];
};

Array<int, 16> arr;
```

这里 `Size = 16`，所以编译器生成的类型中包含：

```cpp
int data[16];
```

它的用途：

- 表示固定大小数组。
- 让大小进入类型系统。
- 编译期计算和优化。
- 避免运行时存储一个 `size` 成员。

## 9. 为什么以前常说只能是整型参数

更准确地说，它叫“非类型模板参数”，并不永远只能是 `int`。

传统 C++ 中，非类型模板参数主要支持：

- 整型常量，例如 `int`、`std::size_t`、`bool`、`char`
- 枚举
- 指针、引用等满足要求的编译期实体

现代 C++ 放宽了限制。C++20 开始支持更多结构化的字面量类型作为非类型模板参数。

但在入门阶段，你可以先记住：非类型模板参数必须是编译期能确定的值，最常见的是整数。

## 10. `Array<int, 16> arr;` 是静态内存还是动态内存

这取决于 `arr` 这个对象定义在哪里。

如果写在函数内部：

```cpp
void f() {
    Array<int, 16> arr;
}
```

`arr` 通常是自动存储期对象，通常放在栈上。`data[16]` 是 `arr` 对象的一部分，也一起在栈上。

如果写在全局作用域：

```cpp
Array<int, 16> arr;
```

`arr` 是静态存储期对象，通常放在全局/静态存储区。`data[16]` 仍然是对象的一部分。

如果你用 `new`：

```cpp
auto* p = new Array<int, 16>;
```

对象在动态存储区，也就是堆上。`data[16]` 仍然是这个对象的一部分。

关键结论：

`T data[Size];` 本身不是动态分配。它不会调用 `new`，也不需要 `delete`。它在哪里，取决于包含它的对象在哪里。

## 11. 如何运行

如果你的环境有 CMake：

```bash
cmake -S . -B build
cmake --build build
./build/template_base
```

也可以直接用 g++：

```bash
g++ -std=c++17 -Wall -Wextra -pedantic src/main.cpp -o template_base
./template_base
```
