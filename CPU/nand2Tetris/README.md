# From Nand to Tetris 学习说明

## 这个项目有没有学习的必要？

有，而且很值得，但前提是你知道自己为什么学。

`From Nand to Tetris` 的价值不在于“学会造一台真的能商用的电脑”，而在于它会把下面这些平时容易割裂的知识连成一条线：

- 从最基础的逻辑门开始，理解计算机“为什么能工作”
- 从硬件设计一路走到汇编、虚拟机、编译器和操作系统
- 建立“抽象分层”的意识，知道每一层解决什么问题
- 以后再学 C/C++、操作系统、编译原理、计算机体系结构时，会明显更扎实

如果你现在的目标只是“尽快找工作写业务”，它不是短期收益最高的内容。

但如果你的目标是下面这些之一，它就非常值得学：

- 想真正理解计算机底层，而不只是会调用工具
- 想补齐计组、编译、系统这几门课之间的断层
- 想提升写高性能程序、系统程序、底层程序时的直觉
- 想为以后学习 C/C++、Rust、操作系统、编译器打基础
- 想建立完整的计算机知识地图

一句话总结：  
这不是“立刻变现”的项目，但它是“长期回报非常高”的项目。

---

## 建议你带着什么目的去学？

不建议把它当成“打卡式通关项目”，建议带着下面 3 个目标去学：

### 1. 建立完整知识地图

你的目标不是记住每个细节，而是回答这些问题：

- 逻辑门如何组成 ALU 和寄存器？
- CPU 如何执行指令？
- 汇编语言和机器指令是什么关系？
- 高级语言为什么能被翻译成机器可执行的东西？
- 操作系统到底帮程序做了什么？

### 2. 训练抽象能力

这个项目最重要的能力不是“搭电路”，而是：

- 理解分层
- 接受隐藏细节
- 在接口稳定的前提下逐层构建系统

这是写大型软件、做工程设计、看源码时非常核心的能力。

### 3. 为后续进阶做准备

学完后，你应该能更自然地进入这些方向：

- C / C++ / Rust
- 计算机组成原理
- 操作系统
- 编译原理
- 数据结构与算法的底层实现
- 高性能计算和性能优化

## 建议的学习路径

不要把它当成“一个连续的超长任务”，而要把它拆成 4 个阶段。

### 第一阶段：硬件基础

内容：

- Boolean Logic
- ALU
- Register / RAM / PC
- CPU
- Memory
- Computer

这一阶段你要关注的不是“电路图画对了没”，而是：

- 组合逻辑和时序逻辑的区别
- 指令是如何驱动数据流的
- CPU、内存、寄存器之间如何协作

学完这一阶段，你应该掌握：

- 能解释一台最简计算机是怎么执行一条指令的
- 能理解加法器、ALU、寄存器、程序计数器的作用
- 能建立“硬件也是模块化系统”的认知

### 第二阶段：机器语言与汇编

内容：

- Machine Language
- Assembler

这一阶段的目标是打通“硬件”与“程序”之间的连接。

学完这一阶段，你应该掌握：

- 机器指令、汇编指令、符号之间的关系
- 程序跳转、条件判断、变量存储在底层如何实现
- 为什么汇编器本质上是在做“翻译”

### 第三阶段：虚拟机与高级语言过渡

内容：

- Virtual Machine I
- Virtual Machine II

这是很多人第一次真正理解“抽象层”的地方。

这一阶段建议重点思考：

- 为什么高级语言不能直接映射到底层机器？
- 为什么需要 VM 作为中间层？
- 调用函数、保存局部变量、处理返回值的本质是什么？

学完这一阶段，你应该掌握：

- 栈式虚拟机的工作方式
- 函数调用栈的基本机制
- 高级语言运行时的一部分核心思想

### 第四阶段：编译器与操作系统雏形

内容：

- Compiler I
- Compiler II
- Operating System

这是整个课程最有“系统感”的部分。

这一阶段你会把一种高级语言逐步翻译成 VM 指令，并补上一套最小操作系统支持。

学完这一阶段，你应该掌握：

- 词法分析、语法分析、代码生成的基本流程
- 编译器并不神秘，本质是“分阶段翻译”
- 操作系统最基础的一些职责：内存、输入输出、屏幕、字符串、数学库等支持

如果学得比较扎实，学完后你应该至少具备下面这些能力。

- 能把一个复杂系统拆成模块逐步实现
- 能围绕接口进行开发，而不是一开始陷入全部细节
- 能更冷静地看待编译器、虚拟机、操作系统这些“大东西”

### 3. 认知上的提升

- 看底层知识不再只剩零散概念
- 学后续课程时更容易建立联系
- 写程序时会更理解“成本、表示、执行、内存、调用”这些问题

## 学习笔记：bit、byte 和 16-bit bus 的关系

这一部分是学 `nand2tetris` 时最基础、也最容易混掉的概念。

### 1. bit 是最小单位

`bit` 就是一个二进制位，只能表示两种状态：

- `0`
- `1`

你可以把它理解成一根线上的电信号状态，或者一个开关的开和关。

所以：

- `1 bit` 只能表示 2 种情况
- `n bit` 可以表示 `2^n` 种情况

例如：

- `2 bit` 可以表示 4 种情况
- `8 bit` 可以表示 256 种情况
- `16 bit` 可以表示 65536 种情况

### 2. byte 是更常用的数据单位

`byte` 是一组 bit。

通常规定：

- `1 byte = 8 bits`

所以：

- `2 bytes = 16 bits`
- `4 bytes = 32 bits`
- `8 bytes = 64 bits`

在很多系统里，一个 byte 常常用来表示：

- 一个字符的一部分编码
- 一个较小范围的整数
- 内存中的最小可寻址单元之一

### 3. 16-bit 的意思

`16-bit` 的核心意思是：

- 某个数据宽度是 16 位
- 或者某个部件一次能处理 16 个 bit

在 `nand2tetris` 里，很多核心部件都是 `16-bit` 的，比如：

- ALU
- 寄存器
- CPU 中处理的数据
- 总线

这意味着这些部件一次面对的不是单独一个 `0/1`，而是一整组 `16` 个 bit。

例如一个 16-bit 数据可以写成：

```text
0000000000001010
```

它一共有 16 位，也就是：

- `16 bits`
- `2 bytes`

### 4. bus 是什么

`bus` 可以理解成“成组的数据通道”。

如果一根线只能传 1 个 bit，那么一个 `16-bit bus` 就相当于：

- 有 16 根并排的线
- 这 16 根线一起传输一个 16-bit 数据

所以 `bus` 关注的是“宽度”和“传输能力”。

例如：

- `1-bit bus` 一次传 1 位
- `8-bit bus` 一次传 8 位
- `16-bit bus` 一次传 16 位

### 5. bit、byte、16-bit bus 之间的关系

可以直接记成下面这几句：

- `bit` 是最小的信息单位
- `byte` 是 8 个 bit
- `16-bit` 表示 16 个 bit 的宽度
- `16-bit bus` 表示一次可以并行传输 16 个 bit
- `16-bit bus = 一次传输 2 bytes`

也就是说：

```text
1 byte = 8 bits
2 bytes = 16 bits
16-bit bus = 一次搬运 16 bits = 2 bytes
```

### 6. 在 nand2tetris 里怎么理解最合适

在这门课里，你可以这样想：

- 单个 `bit` 用来表达最基础的开关状态
- 多个 `bit` 组合起来表示一个数或一条信息
- `16-bit` 是这台 Hack 计算机的基本数据宽度
- `16-bit bus` 是这台机器内部搬运数据的“道路”

所以当你看到：

- `And16`
- `Mux16`
- `Not16`
- `ALU`

本质上都可以理解为：

- 它们不是在处理一个 bit
- 而是在同时处理 16 个 bit 组成的数据

### 7. 一个直观例子

假设寄存器里有一个值：

```text
0000000000000011
```

这是一个 `16-bit` 二进制数，十进制值是 `3`。

如果 CPU 通过一条 `16-bit bus` 把它送到 ALU，那么含义就是：

- 总线上的 16 根线同时承载这 16 个 bit
- ALU 一次性接收到完整的 16-bit 数据

而不是：

- 先传第 1 位
- 再传第 2 位
- 一位一位慢慢传

在这个抽象里，更重要的是“宽度一致、整体传输”。

### 8. 现阶段你至少要记住什么

学到现在，最值得先记住的是：

- `bit` 是 0 或 1
- `byte` 是 8 个 bit
- `16 bits = 2 bytes`
- `16-bit bus` 表示一次可以传 16 个 bit
- `nand2tetris` 里的 Hack 计算机主要按 16-bit 宽度处理数据

如果这几句你已经很稳，后面看寄存器、ALU、内存和 CPU 时就会轻松很多。
